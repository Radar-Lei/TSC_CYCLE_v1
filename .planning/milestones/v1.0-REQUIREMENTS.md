# Requirements: TSC-CYCLE

**Defined:** 2026-02-09
**Core Value:** 给定交叉口实时交通状态,大模型输出的信号配时方案能最大化车辆通过量、最小化排队车辆数

## v1 Requirements

### SFT 数据

- [x] **SFT-01**: 从 train.jsonl 均匀抽取 100 条样本(覆盖不同场景、交叉口、饱和度分布) — ✓ v1.0
- [x] **SFT-02**: 每条样本包含中文短思考(50-200 token),分析各相位饱和度并推算合理的 final 绿灯时长 — ✓ v1.0
- [x] **SFT-03**: 输出格式为 `<think>...<think><solution>[{"phase_id": <int>, "final": <int>}, ...]<solution>` — ✓ v1.0
- [x] **SFT-04**: final 值满足硬约束(min_green ≤ final ≤ max_green,整数秒) — ✓ v1.0
- [x] **SFT-05**: 生成的 SFT 数据保存为 JSONL 格式,供 SFT 训练脚本使用 — ✓ v1.0

### SFT 训练

- [x] **SFTT-01**: Docker shell 脚本(docker/sft_train.sh)遵循 data.sh 的运行模式 — ✓ v1.0
- [x] **SFTT-02**: Python 训练脚本使用 unsloth 加载 Qwen3-4B-Base 并进行 LoRA 微调 — ✓ v1.0
- [x] **SFTT-03**: 自定义 chat template 定义 `<think>...<think><solution>...<solution>` 标签 — ✓ v1.0
- [x] **SFTT-04**: SFT 后的模型权重保存到 outputs/sft/model 目录 — ✓ v1.0

### GRPO 数据

- [x] **GRPD-01**: 从 train.jsonl 提取 GRPO 训练集(prompt + state_file),不包含思考过程 — ✓ v1.0
- [x] **GRPD-02**: prompt 包含 system(角色设定)和 user(交通状态+任务说明)两部分 — ✓ v1.0

### GRPO 训练

- [x] **GRPT-01**: Docker shell 脚本(docker/grpo_train.sh)遵循 data.sh 的运行模式 — ✓ v1.0
- [x] **GRPT-02**: Python 训练脚本加载 SFT 后的模型继续 GRPO 训练 — ✓ v1.0
- [x] **GRPT-03**: 格式正确性 reward:正则匹配 `<think>...<think><solution>...<solution>` 结构 — ✓ v1.0
- [x] **GRPT-04**: SUMO 仿真 reward:loadState → 执行大模型方案(设置相位+时长)→ 统计车辆通过量和排队车辆数 — ✓ v1.0
- [x] **GRPT-05**: think 长度 reward:对过短(<50 token)或过长(>200 token)的思考进行惩罚 — ✓ v1.0
- [x] **GRPT-06**: 多进程并行 reward 计算(每个候选方案独立启动 SUMO 进程) — ✓ v1.0
- [x] **GRPT-07**: GRPO 后的模型权重保存到 outputs/grpo/model 目录 — ✓ v1.0
