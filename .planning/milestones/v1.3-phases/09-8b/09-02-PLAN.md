1. 确认 SFT 模型存在：`test -f outputs/sft/qwen3-8b/model/config.json`

```bash
./docker/grpo_simple_train.sh --config config/config_8b.json
```

这是一个长时间运行的 GPU 训练任务（max_steps=10000，可能需要数小时甚至更长），使用 `run_in_background` 参数执行。

训练完成后验证产出：
1. 检查 outputs/grpo_simple/qwen3-8b/model/ 目录下是否存在 config.json、tokenizer.json、tokenizer_config.json
2. 检查是否存在 .safetensors 权重文件
3. 检查训练日志无 OOM 或其他错误

注意：
- GRPO 使用 SFT 产出的 outputs/sft/qwen3-8b/model 作为基座
- learning_rate=5e-6（比 SFT 的 2e-4 小很多，这是强化学习的典型设置）
- 如果训练提前终止（如 loss 异常），检查 checkpoints 目录是否有可用检查点

按顺序执行三次 GGUF 转换（串行，避免 GPU 资源竞争）：

```bash
# 1. 导出 F16
echo y | ./docker/convert_gguf.sh --model-path outputs/grpo_simple/qwen3-8b/model --outtype f16

# 2. 导出 Q8_0
echo y | ./docker/convert_gguf.sh --model-path outputs/grpo_simple/qwen3-8b/model --outtype Q8_0

# 3. 导出 Q4_K_M
echo y | ./docker/convert_gguf.sh --model-path outputs/grpo_simple/qwen3-8b/model --outtype Q4_K_M
```

注意：用 `echo y |` 管道自动确认覆盖已有文件的交互式提示。

```bash
mkdir -p outputs/grpo_simple/qwen3-8b/gguf
```
