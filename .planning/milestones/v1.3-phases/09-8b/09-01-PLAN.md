  <read_first>docker/convert_gguf.sh</read_first>
  <action>
注意：convert_gguf.sh 默认输出到模型目录内（model-{outtype}.gguf），但需求要求输出到 gguf/ 子目录。需要先创建 gguf 目录，然后在每次转换后将文件移动到正确位置。或者直接用 --model-path 指向 gguf 目录（不行，模型文件在 model/ 目录下）。

实际方案：convert_gguf.sh 会输出到 `{model-path}/model-{outtype}.gguf`，即 `outputs/sft/qwen3-8b/model/model-{outtype}.gguf`。转换完成后手动移动到 gguf/ 目录。

按顺序执行三次转换（每次都需要 GPU 资源，串行执行）：

```bash
# 1. 先导出 F16（作为 Q4_K_M 和 Q8_0 的基础也可单独保留）
./docker/convert_gguf.sh --model-path outputs/sft/qwen3-8b/model --outtype f16

# 2. 导出 Q8_0
./docker/convert_gguf.sh --model-path outputs/sft/qwen3-8b/model --outtype Q8_0

# 3. 导出 Q4_K_M
./docker/convert_gguf.sh --model-path outputs/sft/qwen3-8b/model --outtype Q4_K_M
```

注意：convert_gguf.sh 中如果输出文件已存在会交互式询问是否覆盖。需要用 `echo y |` 管道或在脚本执行前确保文件不存在。

转换完成后，创建 gguf 目录并移动文件：
```bash
mkdir -p outputs/sft/qwen3-8b/gguf
mv outputs/sft/qwen3-8b/model/model-f16.gguf outputs/sft/qwen3-8b/gguf/model-F16.gguf
mv outputs/sft/qwen3-8b/model/model-Q8_0.gguf outputs/sft/qwen3-8b/gguf/model-Q8_0.gguf
mv outputs/sft/qwen3-8b/model/model-Q4_K_M.gguf outputs/sft/qwen3-8b/gguf/model-Q4_K_M.gguf
```
