# Milestone v1.1: 简化版 GRPO 训练

**Status:** ✅ SHIPPED 2026-04-02  
**Phases:** 4-5  
**Total Plans:** 2

## Overview

本里程碑在现有 SFT 模型之上补齐了一条不依赖 SUMO 仿真的简化版 GRPO 流程。重点不在 benchmark 结果，而在于先把 reward、数据生成、训练入口和输出目录全部隔离出来，形成可继续迭代的最小闭环。

## Phases

### Phase 4: 简化版 Reward 与解析核心

**Goal**: 完成可复用的 completion 解析、格式/约束 reward 和饱和度比例 reward，不再依赖 SUMO 仿真。  
**Depends on**: Phase 3  
**Plans**: 1 plan

Plans:

- [x] 04-01: 简化版 reward 与解析核心

**Details:**
- 从 prompt 解析 `phase_waits[*].pred_saturation/min_green/max_green`
- 复用现有 `<end_working_out><SOLUTION>...</SOLUTION>` 协议和 solution JSON 结构
- 对 phase 顺序错误、越界、JSON 错误 completion 做降分
- 目标绿灯时间按 `round(max_green * pred_saturation)` 计算并裁剪

### Phase 5: 简化版训练入口与验证

**Goal**: 基于 Unsloth Docker 提供独立的简化版 GRPO 数据生成、训练入口、输出目录和基础测试。  
**Depends on**: Phase 4  
**Plans**: 1 plan

Plans:

- [x] 05-01: 简化版训练入口与验证

**Details:**
- 从 `outputs/data/train.jsonl` 直接生成 `outputs/grpo_simple/grpo_train.jsonl`
- 新建 `src/grpo_simple/train.py` 与 Docker 包装脚本
- checkpoints 与模型输出统一写入 `outputs/grpo_simple/`
- 保持旧版 `src/grpo/`、`docker/grpo_train.sh` 与 `outputs/grpo/` 不受影响

---

## Milestone Summary

**Key Decisions:**

- 简化版 GRPO 新建 `src/grpo_simple/`，而不是直接侵入旧版 `src/grpo/`
- reward 降维到“格式 + 约束 + 饱和度比例”最小闭环，优先去掉 SUMO 外部依赖
- 训练数据回到原始 `outputs/data/train.jsonl`，避免沿用不匹配新目标的 SFT 样本

**Issues Resolved:**

- reward 从 SUMO baseline 解耦
- 训练输出目录与旧版 GRPO 完全隔离
- 数据生成和训练入口都统一到 Unsloth Docker

**Issues Deferred:**

- 完整长时 GRPO 训练与效果评估
- benchmark 回接与 reward 调参

**Technical Debt Incurred:**

- 训练入口仍依赖 GPU 可见的 Docker/Unsloth 环境，后续若需要更强自动化验证，可再补 dry-run 能力
