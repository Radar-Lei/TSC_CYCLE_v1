  <action>
    用户需要在 DGX-Spark 上依次执行以下三条命令导出 GGUF 模型:

    **注意执行顺序**: 先导出 f16（因为 Q4_K_M 和 Q8_0 的二次量化也需要先生成 f16 中间文件，但脚本会自动处理并删除中间文件。为了保留 f16 最终产物，先单独导出 f16）。

    ```bash
    # 1. 导出 F16 (EXPORT-03)
    chmod +x docker/convert_gguf.sh
    ./docker/convert_gguf.sh --model-path outputs/sft/model --outtype f16

    # 2. 导出 Q8_0 (EXPORT-02) — 会生成临时 f16 中间文件再量化，最终删除中间文件
    ./docker/convert_gguf.sh --model-path outputs/sft/model --outtype Q8_0

    # 3. 导出 Q4_K_M (EXPORT-01) — 同上
    ./docker/convert_gguf.sh --model-path outputs/sft/model --outtype Q4_K_M
    ```

    每次执行注意:
    - 脚本会 git clone llama.cpp (shallow clone)，第一次较慢
    - Q8_0 和 Q4_K_M 需要编译 llama-quantize 工具，会花额外时间
    - 如果提示输出文件已存在，输入 y 确认覆盖
    - 每次完成后会打印文件大小

    预期文件大小 (Qwen3-4B):
    - f16: ~8GB
    - Q8_0: ~4GB
    - Q4_K_M: ~2.5GB
