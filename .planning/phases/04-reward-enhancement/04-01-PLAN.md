**修改 `_run_sumo_evaluation()` 函数，新增 delay 采集：**

1. 在仿真循环（`for _ in range(duration)` 每个 phase 的步进）中，每步遍历 `controlled_lanes` 上的车辆，累加 `traci.vehicle.getWaitingTime(vid)`
2. 注意使用 set 跟踪已计数车辆避免重复：每步获取当前 lane 上所有车辆 ID，对每个车辆累加当步的 waitingTime
3. 实际上 getWaitingTime 返回的是该车辆当前连续等待的秒数，所以每步都累加所有车辆的 waitingTime 就是总延误
4. 返回值新增 `total_delay` 字段：`{"passed_vehicles": N, "queue_vehicles": N, "total_delay": float}`

**重写 `sumo_simulation_reward()` 中的分数计算逻辑（第 481-501 行区域）：**

1. 从 config 读取新字段：
   - `throughput_weight = _config["sumo_throughput_weight"]`（已有）
   - `queue_weight = _config["sumo_queue_weight"]`（已有）
   - `delay_weight = _config["sumo_delay_weight"]`（新增）
   - `negative_ratio = _config["sumo_negative_ratio"]`（新增）

2. 对每个完成仿真的样本，计算三维改善率：
   ```python
   # Throughput: 越多越好
   if baseline_passed > 0:
       t_imp = (model_passed - baseline_passed) / baseline_passed
   else:
       t_imp = 0.0  # baseline=0 时跳过该维度

   # Queue: 越少越好
   if baseline_queue > 0:
       q_imp = (baseline_queue - model_queue) / baseline_queue
   else:
       q_imp = 0.0

   # Delay: 越少越好
   baseline_delay = baseline.get("total_delay", 0)
   model_delay = result["total_delay"]
   if baseline_delay > 0:
       d_imp = (baseline_delay - model_delay) / max(baseline_delay, 1)
   else:
       d_imp = 0.0
   ```

3. 加权求和：
   ```python
   raw_score = throughput_weight * t_imp + queue_weight * q_imp + delay_weight * d_imp
   ```

4. 非线性压缩 + 负分控制：
   ```python
   import math
   if raw_score >= 0:
       # 正分：log(1+x) 压缩，然后缩放到 [0, sumo_max_score]
       # log(1+x) 对小值近似线性，对大值压缩
       # 缩放系数：让 raw_score=1.0（即 100% 改善）映射到约 sumo_max_score * 0.7
       compressed = math.log(1 + raw_score)
       # log(2) ≈ 0.693, 所以 100% 改善 -> 0.693 -> 归一化后约 0.7 * sumo_max_score
       score = compressed / math.log(2) * sumo_max_score
       # 仍然不设上限 cap，但 log 自然压缩了大值
   else:
       # 负分：线性映射，下界为 -sumo_max_score * negative_ratio
       floor = -sumo_max_score * negative_ratio
       score = max(raw_score * sumo_max_score, floor)
   ```

5. 更新 debug 打印：增加 delay 相关信息和改善率
