在现有 test_rewards.py 中新增 SUMO reward 分布验证功能。保留现有的 L1/L2/think 格式测试，新增以下内容：

**1. 新增 `validate_sumo_distribution()` 函数：**

- 接收参数：`config_path`（str）、`sample_size`（int，默认 100）
- 从 config 加载 grpo_data_dir 路径，读取 grpo_train.jsonl
- 随机抽样 `sample_size` 条样本（如果总数不足则全用）。使用分层抽样：按 scenario（从 state_file 路径解析）分组，每组按比例抽样
- 对每条样本：
  a. 从 prompt 中解析 phase_waits
  b. 构造一个"格式正确、约束满足"的 completion（使用 phase_waits 中的 min_green 作为 final 值，保证通过 L1+L2 gate）
  c. 调用 `sumo_simulation_reward(prompts=[prompt], completions=[completion], state_file=[sf], tl_id=[tl])`
- 收集所有 SUMO reward 分数

**2. 统计输出：**
```
========== SUMO Reward 分布验证 ==========
样本数: N
均值:   X.XXX
标准差: X.XXX
最小值: X.XXX
最大值: X.XXX
分位数:
  10%: X.XXX
  25%: X.XXX
  50%: X.XXX
  75%: X.XXX
  90%: X.XXX
唯一值数量: N
负分比例: XX.X%
零分比例: XX.X%
==========================================
```

**3. 自动检查规则（`check_distribution_quality()`）：**
- 标准差下界：`std >= 0.5`（如果 reward 的 std 低于 0.5，说明区分度不够）
- 唯一值数量下界：`unique_values >= sample_size * 0.3`（至少 30% 的样本有不同分数）
- 非零比例：`non_zero_ratio >= 0.5`（至少 50% 的样本得到非零分数）
- 返回 `(passed: bool, reasons: list[str])`
- 如果不通过，打印红色警告和具体原因

**4. 新增命令行参数支持：**
- `--sumo-validate`：运行 SUMO 分布验证（默认不运行，因为需要 SUMO 环境）
- `--sample-size N`：验证样本数（默认 100）
- 无参数时：只运行现有的格式测试（保持向后兼容）

**5. main() 函数改写：**
- 使用 argparse 处理命令行参数
- `--sumo-validate` 模式下：
  a. 调用 `init_rewards()` 初始化
  b. 运行现有格式测试
  c. 调用 `validate_sumo_distribution()`
  d. 调用 `check_distribution_quality()`
  e. 如果检查不通过，`sys.exit(1)`
- 无参数时：只运行现有格式测试（不需要 SUMO）

**注意：**
- SUMO 验证需要在 Docker 容器中运行（有 SUMO 环境）
- 使用 import 时注意路径（在 Docker 中工作目录是 /home/samuel/SCU_TSC）
- sumo_simulation_reward 函数需要 init_rewards 先初始化

在 `docker/grpo_train.sh` 中，在现有的"通过 Docker 容器执行 GRPO 训练"命令之前，新增一个 reward 验证步骤：

**1. 在 "前置检查" 之后、"开始 GRPO 训练" 之前，新增验证段：**

```bash
# Reward 分布验证
echo ""
echo "[验证] 运行 reward 分布验证..."
docker run --rm \
    --gpus all \
    --shm-size=32GB \
    --user "$(id -u):$(id -g)" \
    -v "${PROJECT_DIR}:${CONTAINER_WORKDIR}:rw" \
    -w "${CONTAINER_WORKDIR}" \
    -e SUMO_HOME=/usr/share/sumo \
    -e HF_HOME="${CONTAINER_WORKDIR}/.cache/huggingface" \
    --entrypoint python3 \
    "${IMAGE_NAME}" \
    -m src.grpo.test_rewards \
        --sumo-validate \
        --sample-size 50

VALIDATE_EXIT=$?
if [ ${VALIDATE_EXIT} -ne 0 ]; then
    echo ""
    echo "[错误] Reward 分布验证未通过！"
    echo "请检查 reward 公式和 baseline 配置。"
    echo "如需跳过验证，使用 --skip-validate 参数。"
    exit 1
fi
echo "[验证] Reward 分布验证通过"
echo ""
```

**2. 新增 `--skip-validate` 参数支持：**
- 在脚本开头解析参数，检查是否有 `--skip-validate`
- 如果有，跳过验证步骤
- 将其他参数传递给训练命令
- 实现方式：
  ```bash
  SKIP_VALIDATE=false
  TRAIN_ARGS=()
  for arg in "$@"; do
      if [ "$arg" = "--skip-validate" ]; then
          SKIP_VALIDATE=true
      else
          TRAIN_ARGS+=("$arg")
      fi
  done
  ```
- 验证段用 `if [ "$SKIP_VALIDATE" = "false" ]; then ... fi` 包裹
- 训练命令改用 `"${TRAIN_ARGS[@]}"` 传参

**3. 使用样本数 50（而非 100）以控制验证时间：**
- Docker 环境中 SUMO 仿真较慢，50 条足够验证分布趋势
- 用户可以通过修改脚本调整
