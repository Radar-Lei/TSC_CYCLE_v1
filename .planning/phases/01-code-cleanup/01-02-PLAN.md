```python
def main():
    args = parse_args()

    # 加载配置
    config = load_config(args.config) if os.path.exists(args.config) else {}
    sim_config = config.get('simulation', {})
    paths_config = config.get('paths', {})

    # 确定参数（命令行 > config.json）
    environments_dir = args.environments_dir or paths_config.get('environments_dir', 'sumo_simulation/environments')
    workers = args.workers or sim_config.get('parallel_workers', None)
    warmup_steps = args.warmup_steps or sim_config.get('warmup_steps', 300)
    output_dir = os.path.abspath(args.output_dir)
    state_dir = os.path.abspath(args.state_dir)

    # 发现场景
    scenarios = discover_environments(os.path.abspath(environments_dir))
    if not scenarios:
        print(f"错误: 未找到有效场景")
        sys.exit(1)

    # 阶段 1: 为每个场景生成 phase_config（串行，轻量）
    # 阶段 2: 展开所有 (场景, 交叉口) 对为任务列表
    # 阶段 3: 使用 Pool 并行消费任务
    # 阶段 4: 合并结果到 train.jsonl
```

**核心逻辑 - 扁平任务池:**

```
1. discover_environments() 获取所有场景
2. 对每个场景:
   a. 生成 phase_config_<scenario>.json（如不存在）
   b. 从 phase_config 读取该场景的所有交叉口 ID
   c. 每个交叉口生成一个任务 = (scenario_name, rou_file, sumocfg, phase_config_path, tl_id, output_dir, state_dir, warmup_steps)
3. 所有任务放入统一列表
4. Pool(workers).imap_unordered(worker_fn, tasks) 并行消费
5. 每个 worker 调用 DaySimulator（target_tl_ids=[单个 tl_id]）
6. Fail-fast: 任一任务失败立即终止 Pool 并报错退出
7. 合并所有成功结果到 train.jsonl
```

**Worker 函数:**

```python
def _simulate_intersection(args):
    """单个交叉口的仿真 worker"""
    task_id, scenario_name, rou_file, config = args

    from src.data_generator.day_simulator import DaySimulator

    simulator = DaySimulator(task_id, rou_file, config)
    result = simulator.run()
    result['scenario_name'] = scenario_name
    return result
```

**Fail-fast 模式:**
当某个 worker 返回 `status == 'error'` 时，立即 `pool.terminate()` 并打印错误信息后 `sys.exit(1)`。

- `--workers`: 并行进程数

- `--dry-run`: Dry-run 模式
