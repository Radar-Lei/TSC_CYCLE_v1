当前的 run.sh 是一个 Docker 构建和运行入口。按用户决策，run.sh 应该串联执行全流程（data -> sft -> grpo）。Docker 构建/运行的职责交由 Dockerfile 和 `docker run` 命令本身处理。

- `#!/usr/bin/env bash` + `set -euo pipefail`
- 设置 `SCRIPT_DIR` 和 `PROJECT_DIR`

- 任一步骤失败则立即停止（由 set -e 保证）
- 简洁输出：总流程开始/结束，各阶段状态
- 不依赖 lib/*.sh 或其他辅助脚本

**2. 统一输出路径确认:**

确认所有脚本中的输出路径指向 `outputs/` 目录：
- 数据: `outputs/data/`

- SFT: `outputs/sft/`
- GRPO: `outputs/grpo/`
