  <name>Task 1: 创建增强版数据生成脚本</name>
  <files>src/scripts/generate_enhanced_sft_data.py</files>
  <action>
创建 Python 脚本 `src/scripts/generate_enhanced_sft_data.py`，功能：
1. 读取现有的 `outputs/sft/think_workspace.jsonl`
2. 对每条数据，扩展 think 字段内容：
   - 目标长度：200-300 中文字符（约 300-400 token）
   - 扩展内容包含：交通状态分析、各相位饱和度解读、配时决策推理、约束满足验证
   - 均衡扩展思考链各环节，不侧重某一特定部分
   - 自由生成风格，不做结构化模板约束
3. 生成规则：
   - phase_waits[i].pred_saturation > 1.0：高饱和度，应分配接近 max_green
   - pred_saturation 在 0.5-1.0：中等饱和度，按比例分配
   - pred_saturation < 0.5：低饱和度，可分配 min_green 或略高
   - 必须明确说明每个 phase 的 final 值计算依据
4. 备份原有文件：将 think_workspace.jsonl 复制为 think_workspace.jsonl.bak
5. 串行逐条处理，每条数据生成后立即保存到新文件
6. 调用 generate_sft_data.py assemble 子命令生成最终 sft_train.jsonl

关键实现点：
- 使用 argparse 支持命令行参数：--workspace（输入）、--output（输出）、--samples（原始样本）
- 日志输出：处理进度、生成的 think 长度统计
- 错误处理：JSON 解析失败时记录错误并跳过
  </action>

<task type="auto">
  <name>Task 2: 创建数据校验脚本</name>
  <files>src/scripts/validate_sft_data.py</files>
  <action>
创建 Python 脚本 `src/scripts/validate_sft_data.py`，功能：
1. 校验 JSON 结构：
   - 每条数据包含 messages 数组
   - messages 包含 system、user、assistant 三个角色
   - assistant content 包含 <start_working_out>...<end_working_out><SOLUTION>...</SOLUTION> 格式
2. 校验标签格式：
   - 检测 <start_working_out> 和 <end_working_out> 成对出现
   - 检测 <SOLUTION> 和 </SOLUTION> 成对出现
   - 提取 SOLUTION 内容并验证是有效 JSON 数组
3. 校验约束满足：
   - 解析 SOLUTION JSON，获取各 phase_id 和 final 值
   - 验证每个 final 满足对应的 min_green <= final <= max_green
4. 统计 token 长度：
   - 使用 tiktoken 或简单估算（中文约 0.5 token/字符）
   - 报告思考链长度的 min/max/mean/median
5. 输出：
   - 通过/失败统计
   - 错误详情（行号、错误类型）
   - 长度分布统计

命令行参数：--input（输入文件）、--verbose（详细输出）
  </action>

  </done>
