  <action>
    在 generate_training_data.py 中添加 CoT 格式转换功能，实现 DATA-07（二度生成）。

    1. 添加 `convert_to_sft_format(raw_jsonl_path, sft_jsonl_path)` 函数：
       - 读取原始 train.jsonl（每行包含 prompt, prediction, state_file, metadata）
       - 转换为 SFT chat 格式：
         ```json
         {
           "messages": [
             {"role": "system", "content": "你是交通信号配时优化专家。"},
             {"role": "user", "content": "<原始 prompt 中去掉系统角色行的内容>"},
             {"role": "assistant", "content": "<think>\n\n</think>\n[{\"phase_id\": N, \"final\": M}, ...]"}
           ]
         }
         ```
       - assistant content 的构建逻辑：
         a. `<think>` 部分：使用空占位符 `<think>\n\n</think>`，不生成任何分析文本。这让模型在训练时自己学习填充推理内容（per phase context 锁定决策："不预生成 CoT"）
         b. JSON 部分：基于 pred_saturation 计算 final 绿灯时间：
            - 饱和度 > 1.0：偏向 max_green
            - 饱和度 < 0.5：偏向 min_green
            - 其他：按饱和度线性插值 min_green 到 max_green
            - final 必须为整数，且满足 min_green <= final <= max_green

    2. 修改 `main()` 函数，在阶段 5（合并 train.jsonl）之后添加阶段 6：
       - 阶段 6：CoT 格式转换
       - 读取 outputs/training/train.jsonl
       - 转换为 SFT chat 格式
       - 输出到 outputs/sft/train.jsonl
       - 打印转换统计（总条数、成功条数）

    3. 确保 outputs/sft/ 目录自动创建

    注意事项：
    - prompt 字段中的系统角色行（"你是交通信号配时优化专家。"）应提取到 messages 的 system role
    - user content 应包含 JSON 输入块和任务描述
    - prediction 字段的 phase_waits 用于生成 assistant 的 JSON 输出
    - state_file 和 metadata 不包含在 SFT 训练数据中（这些是供 GRPO 使用的）
  </action>
