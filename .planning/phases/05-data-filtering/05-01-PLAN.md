<objective>
创建 GRPO 数据过滤脚本和 Docker 入口脚本，从 grpo_train.jsonl 中剔除空交叉口和极低流量样本。

Purpose: 训练数据中约 14% 的样本所有相位 pred_saturation=0（空交叉口），另有约 4% 极低流量，这些样本对 GRPO 训练无贡献（baseline passed=0, queue=0 时 reward 无法产生有意义梯度）。过滤后提升训练效率和数据质量。

Output: 过滤脚本 `src/scripts/filter_grpo_data.py`，Docker 入口脚本 `docker/filter_data.sh`，更新后的 `config/config.json`。
</objective>

**1. 更新 config/config.json**

在 `training.grpo` 下新增 `data_filter` 配置块：

```json
"data_filter": {
  "saturation_sum_threshold": 0.1,
  "input": null,
  "output_suffix": "_filtered",
  "rejected_suffix": "_rejected"
}
```

- `saturation_sum_threshold`: 所有相位 pred_saturation 之和 < 此值的样本被剔除。默认 0.1（数据分析显示 14% 样本 total_sat=0，约 18% < 0.1）
- `input`: null 表示使用默认路径 `{grpo_data_dir}/grpo_train.jsonl`
- `output_suffix`/`rejected_suffix`: 输出文件后缀

**2. 创建 src/scripts/filter_grpo_data.py**

独立 Python 脚本，使用 `python -m src.scripts.filter_grpo_data` 运行。

核心逻辑：
1. 加载 config.json 获取过滤参数和路径
2. 支持 CLI 参数覆盖：`--config`, `--input`, `--output`, `--threshold`
3. 逐行读取 grpo_train.jsonl
4. 对每条记录：用正则从 prompt 最后一条 user message 的 content 中提取 `phase_waits` JSON 数组（与 baseline.py 相同的方式）
5. 计算所有相位 `pred_saturation` 之和
6. 若 `saturation_sum < threshold`：写入 rejected 文件
7. 否则：写入 filtered 输出文件

路径计算逻辑：
- 输入默认：`{grpo_data_dir}/grpo_train.jsonl`
- 输出默认：`{grpo_data_dir}/grpo_train_filtered.jsonl`（基于输入文件名 + output_suffix）
- rejected 默认：`{grpo_data_dir}/grpo_train_rejected.jsonl`（基于输入文件名 + rejected_suffix）

统计报告逻辑：
- 统计所有样本的 saturation_sum 分布
- 打印到终端（中文）：
  ```
  === GRPO 数据过滤报告 ===
  过滤阈值: saturation_sum < 0.1
  输入文件: outputs/grpo/grpo_train.jsonl
  过滤前: 16788 条
  过滤后: 13781 条
  剔除: 3007 条 (17.9%)

  流量分布 (saturation_sum):
    全部样本: min=0.0000 max=5.7468 mean=0.9031 median=0.8110
    保留样本: min=0.1000 max=5.7468 mean=X.XXXX median=X.XXXX
    剔除样本: min=0.0000 max=0.0999 mean=X.XXXX median=X.XXXX

  输出文件:
    保留: outputs/grpo/grpo_train_filtered.jsonl
    剔除: outputs/grpo/grpo_train_rejected.jsonl
  ```
- 同时写入纯文本文件（与 filtered 文件同目录，名为 `grpo_train_filter_report.txt`）

脚本结构：
- `parse_saturation_sum(sample)` -> float: 从单条样本提取 phase_waits 并计算 saturation_sum
- `filter_data(input_path, output_path, rejected_path, threshold)` -> dict: 主过滤函数，返回统计
- `format_report(stats, config_info)` -> str: 格式化统计报告
- `main()`: argparse + config 加载 + 调用过滤 + 输出报告

注意：使用与 baseline.py 相同的正则模式提取 phase_waits：
```python
re.search(r'"phase_waits"\s*:\s*(\[.*?\])', content, re.DOTALL)
```

退出码：0=成功，1=失败（无输入文件等）

创建 `docker/filter_data.sh`，作为 Docker 容器中的入口脚本，串联执行：数据过滤 → baseline 重算。

脚本结构（参照 `docker/grpo_baseline.sh` 的风格）：

```bash
#!/usr/bin/env bash
set -euo pipefail

# 头部注释：功能说明、输出文件、用法
# 用法：
#   ./docker/filter_data.sh                           # 使用默认配置
#   ./docker/filter_data.sh --threshold 0.2           # 覆盖阈值
#   ./docker/filter_data.sh --skip-baseline            # 跳过 baseline 重算

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(cd "${SCRIPT_DIR}/.." && pwd)"

IMAGE_NAME="qwen3-tsc-grpo:latest"
CONTAINER_WORKDIR="/home/samuel/SCU_TSC"
```

**步骤 1：数据过滤**
- 容器名 `grpo-filter`
- 执行 `python3 -m src.scripts.filter_grpo_data --config config/config.json`
- 传递额外参数（`"$@"` 中除 `--skip-baseline` 外的参数）

**步骤 2：Baseline 重算（可选）**
- 检查 `--skip-baseline` 参数，若存在则跳过
- 容器名 `grpo-baseline-filtered`
- 执行 `python3 -m src.grpo.baseline --config config/config.json --input {filtered_path} --output {baseline_path} --workers 16`
- filtered_path 从 config 解析或使用默认路径
- 这样 baseline.json 只包含过滤后数据的 baseline

**参数处理：**
- 解析 `--skip-baseline` 标志（从 `$@` 中提取）
- 其余参数透传给 filter_grpo_data.py

**输出打印：**
- 清晰的中文标题和分隔线（与 grpo_baseline.sh 风格一致）
- 显示项目目录、Docker 镜像名
- 每步完成后打印状态

**Docker 运行参数：**
- `--rm`, `--gpus all`, `--shm-size=32GB`（与 grpo_baseline.sh 一致）
