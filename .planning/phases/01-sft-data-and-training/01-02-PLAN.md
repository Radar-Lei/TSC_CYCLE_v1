  <name>Task 1: 创建 SFT 数据组装与校验脚本（仅 solution 生成 + 数据结构 + 校验）</name>

创建 Python 脚本 src/scripts/generate_sft_data.py，功能如下：

**重要：此脚本不生成 think 内容。** think 内容由 AI 在 Task 2 中逐条手工生成（per user decision: "think 内容由 AI 直接手工生成，不用代码程序生成"）。

脚本提供两个主要功能：

**功能 A：生成 solution 值 + 导出数据框架（子命令 `prepare`）**

1. 读取 outputs/sft/sampled_100.jsonl 中的每条样本
2. 对每条样本，根据 prediction.phase_waits 中的数据，程序化生成 final 值：
   - 基于 pred_saturation 按比例在 [min_green, max_green] 范围内分配
   - saturation 越高的相位分配越接近 max_green，saturation 为 0 的相位分配 min_green
   - 简单线性映射：如果所有相位 saturation 都为 0，都给 min_green
   - 如果只有一个相位有 saturation > 0，该相位给 max_green，其余给 min_green
   - 一般情况：按 saturation 比例在 min-max 范围内线性分配
   - final 必须为整数秒（round 取整）
   - 约束校验：确保 min_green <= final <= max_green（clamp）
3. 输出 solution JSON 数组：[{"phase_id": <int>, "final": <int>}, ...]
4. 将每条样本的关键信息导出为 outputs/sft/think_workspace.jsonl，每行包含：
   ```json
   {
     "index": 0,
     "tl_id": "...",
     "scenario": "...",
     "phase_waits": [{"phase_id": ..., "pred_saturation": ..., "min_green": ..., "max_green": ..., "capacity": ...}, ...],
     "solution": [{"phase_id": ..., "final": ...}, ...],
     "think": ""
   }
   ```
   - think 字段留空（""），由 Task 2 的 AI 填写

**功能 B：组装最终 JSONL（子命令 `assemble`）**

1. 读取 outputs/sft/think_workspace.jsonl（已由 AI 填写 think 内容）
2. 读取 outputs/sft/sampled_100.jsonl（获取原始 prompt）
3. 对每条数据，组装为 SFT 训练格式：
   ```json
   {
     "messages": [
       {"role": "system", "content": "你是交通信号配时优化专家。"},
       {"role": "user", "content": "<原始 prompt 中去掉系统角色行的部分>"},
       {"role": "assistant", "content": "<think>{think_text}<think><solution>{solution_json}<solution>"}
     ]
   }
   ```
   格式细节（per user decision 和 SFT-03）：
   - 使用 `<think>...<think>` 和 `<solution>...<solution>` 标签（重复开标签作为关闭标签）
   - system content = "你是交通信号配时优化专家。"
   - user content = 原始 prompt 减去第一行系统角色行（即从【cycle_predict_input_json】开始）
   - assistant content = `<think>{think_text}<think><solution>{solution_json}<solution>`

4. 双重校验（per user decision）：
   - 校验每条数据的 final 值满足 min_green <= final <= max_green
   - 校验 phase_id 顺序和数量一致
   - 校验 think 非空且长度合理（不为空字符串）
   - 校验 messages 结构完整（3 个角色）

5. 输出到 outputs/sft/sft_train.jsonl
6. 打印统计：总条数、think 平均长度、约束违反数（应为 0）、空 think 数（应为 0）

命令行接口：
```
# Step 1: 生成 solution + 导出工作区文件
python3 -m src.scripts.generate_sft_data prepare --input outputs/sft/sampled_100.jsonl --output outputs/sft/think_workspace.jsonl

# Step 2 (AI 手工填写 think 后): 组装最终数据
python3 -m src.scripts.generate_sft_data assemble --workspace outputs/sft/think_workspace.jsonl --samples outputs/sft/sampled_100.jsonl --output outputs/sft/sft_train.jsonl
```

  <name>Task 2: AI 逐条生成 think 内容并组装最终 SFT 数据</name>
  <files>outputs/sft/think_workspace.jsonl, outputs/sft/sft_train.jsonl</files>
  <action>
**此任务由 AI 执行器直接完成，不调用外部 LLM API，AI 直接手工撰写 think 内容（per user decision）。**

流程：

1. 读取 outputs/sft/think_workspace.jsonl（由 Task 1 生成）
2. 逐条阅读每条样本的 phase_waits 和 solution 数据
3. 为每条样本手工撰写 think 内容（中文，50-200 token），写入 think 字段

**think 内容撰写要求：**
- 风格：简洁分析型，中英混合，专业术语可用英文（如 saturation, capacity）
- 内容：分析每个相位的 pred_saturation 大小关系，简要说明分配思路
  - 例如："phase 0 的 saturation 为 0.85，远高于 phase 1 的 0.12，应优先分配更多绿灯时间。phase 2 saturation 为 0，分配 min_green 即可。"
- 不需要数值计算过程（per user decision），只需定性方向分析
- 参考 sample_prompt_result.md 中的 RAW 部分风格
- 每条 think 应有变化，避免完全机械化的模板感
- think 结构：逐相位分析 -> 简要总结分配思路

4. 将填写完 think 的数据写回 outputs/sft/think_workspace.jsonl
5. 运行 assemble 子命令组装最终数据：
```
python3 -m src.scripts.generate_sft_data assemble --workspace outputs/sft/think_workspace.jsonl --samples outputs/sft/sampled_100.jsonl --output outputs/sft/sft_train.jsonl
```

**执行策略（给执行器的提示）：**
- 可以分批处理，每批 10-20 条，读取、撰写、写回
- 每条 think 需要阅读该样本的 phase_waits 数据来决定内容
- 保持语言风格一致但表述多样化
- 完成后运行 assemble 命令并确认校验通过
